# 02. 要素抽出 & 技術選定

調査日: 2026-06-22

## 1. 要素抽出（市場調査 → 機能要件）

市場調査で抽出したコアメカニクスを、MVP に入れるもの／後回しにするものへ仕分けする。

### MVP に含める（Must）

| # | 要素 | 内容 | 根拠（市場調査） |
|---|------|------|------------------|
| F1 | タップ獲得 | 中央のオブジェクトをタップで資源獲得＋フローティング演出 | コアフック「連打の爽快感」 |
| F2 | クリック強化 | 1 タップあたりの獲得量をコストを払って強化 | 「育成で効率化」 |
| F3 | 自動生産ユニット | 複数種の自動生産（所持数で価格が指数上昇） | 「育成で自動化」/ 指数カーブ |
| F4 | 放置報酬 | 離席時間に応じた獲得（上限あり）と復帰モーダル | 「放置で報酬」/ 再訪動機 |
| F5 | 桁表記 | K, M, B, T… のサフィックス表記 | インフレ感の演出 |
| F6 | セーブ/ロード | localStorage に自動保存・自動復元 | 継続プレイの前提 |
| F7 | 転生(プレステージ) | 進行リセットと引き換えに恒久倍率を獲得 | 長期リテンションの要 |

### 追加実装済み（QoL / MVP 後の第1弾）

| # | 要素 | 内容 | 根拠（市場調査） |
|---|------|------|------------------|
| F8 | まとめ買い | ×1 / ×10 / Max を切替。等比数列の和で合計コスト算出、Max は所持額で買える最大数を購入 | 中盤以降の快適性（UI の快適性が選択基準） |
| F9 | タップブースト | 一定時間 全生産・タップが ×2、終了後クールダウン | リワード型ブーストの無課金版。リテンション施策 |
| F10 | 実績 | 11種の達成条件を自動判定。解除ごとに全生産 +1%、トースト通知 | リテンションの王道機能 |
| F11 | デイリーボーナス | 1日1回受取、連続日数で報酬増（最大×7） | 再訪促進（毎日ログインの動機） |
| F12 | 演出強化 | 独自SVGアイコン(絵文字不使用) / WebAudio 効果音 / Vibration 触覚 / クリティカル(5%で×7) / タップ時パーティクル・数値ポップ・購入フラッシュ・達成/デイリーのバースト。設定で ON/OFF | タップの爽快感＝コア体験＋報酬感の強化 |
| F10b | 実績の達成率表示 | 各実績に進捗バーと「現在値/目標値」(例 20/50) を表示 | 達成までの距離が見える＝継続動機 |
| F13 | 恒久アップグレード | ネビュラで購入する4系統(クリック/生産/ブースト/放置)の強化ツリー | 転生通貨の使い道＝深みと長期目標 |

### MVP では入れない（Should / Could / 後続）

- 収益化（IAP・リワード広告） … まずコアループの面白さ検証を優先
- 実績/ミッション、デイリーボーナス、リーダーボード等のソーシャル要素
- BGM/SE、リッチアニメーション、複数ステージ
- プッシュ通知（放置報酬の再訪促進）

## 2. 技術（言語・プラットフォーム）選定 — 論拠

### 候補比較

| 候補 | 長所 | 短所 | MVP 適性 |
|------|------|------|----------|
| **Web (HTML/CSS/JS, PWA)** | ビルド不要・即実行/検証可。1 コードで iOS/Android/PC。PWA でホーム画面設置＆オフライン動作。クリッカーは UI 主体で Web と相性◎ | ストア配信・課金・プッシュ通知はそのままでは弱い | **◎** |
| **Flutter (Dart)** | 1 コードで iOS/Android ネイティブ。UI 表現力高い。ホットリロード | 本環境に SDK 未導入。Android SDK/実機/エミュレータ無しでビルド検証不可 | △ |
| **React Native (JS/TS)** | クロスプラットフォーム。Web 資産流用しやすい | ネイティブビルド環境が必要。MVP には過剰 | △ |
| **ネイティブ (Kotlin/Swift)** | 最高性能・OS 機能フル活用 | iOS/Android 二重開発。MVP には過剰 | × |
| **Unity (C#)** | ゲームエンジン。物理/3D に強い | 2D の数値 UI ゲームには過剰。ビルド重い | × |

### 検証した実行環境（本コンテナ）

```
node    : v22.x   ← 利用可
python3 : 3.11    ← 利用可（静的配信に使用）
java    : 21      ← 利用可
flutter : 未導入  ← ネイティブビルド検証は不可
```

### 決定: **Web (HTML + CSS + Vanilla JavaScript) を PWA として実装**

**論拠:**

1. **「最小限のアプリ」に最適** — ビルドツールチェーン不要で、コアループの面白さ検証に最短到達できる。
2. **本環境で実際に動作検証できる唯一の選択肢** — Flutter/Android SDK は未導入のため、ネイティブビルドは本環境で検証不能。Web なら `python3 -m http.server` で即起動・確認可能。
3. **スマホアプリ要件を満たせる** — レスポンシブ＋PWA（manifest + Service Worker）でホーム画面に「アプリとして」追加でき、オフラインでも起動する。iOS/Android 両対応。
4. **ジャンル特性との適合** — クリッカーは 3D/物理を伴わない UI・数値主体のゲームで、Web/DOM で十分かつ快適に表現できる。
5. **フレームワーク非依存（Vanilla）** — 依存ゼロ＝学習・改修コスト最小。MVP の保守性が高い。

**段階的成長パス（将来）:** ストア配信・課金・プッシュ通知が必要になった段階で、
(a) この PWA を **Capacitor** で薄くネイティブ化、または (b) **Flutter** で再実装、を推奨。
今回のコアロジック（生産・コスト・転生の計算式）は言語非依存なのでそのまま移植できる。

## 3. 受け入れ条件（MVP 完了の定義）

- [ ] タップで資源が増え、フローティング数値が表示される
- [ ] クリック強化・自動生産ユニットを購入でき、価格が指数的に上昇する
- [ ] 自動生産により毎秒資源が増える
- [ ] リロードしても進行が保持される（localStorage）
- [ ] 一定時間後に再訪すると放置報酬が付与される
- [ ] 転生で恒久倍率を獲得し進行がリセットされる
- [ ] スマホ画面でレイアウトが崩れない／PWA としてインストールできる
