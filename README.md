# ASTRAL WARDEN（仮）

タイミングアクションRPGの **戦闘プロトタイプ**。
「腕次第で格上にも勝てる」テクニカルな戦闘の面白さを核に据えた、完全オリジナルのゲームです。
名称・キャラクター・用語・素材はすべて独自に作成しており、既存作品の固有名詞・アセットは一切使用していません。

## 実装済みの戦闘システム（プロトタイプ）

| システム | 内容 |
| --- | --- |
| **EN（エナジー）** | スキルで消費（コストはスキルごと）。**自動回復はしない**。回復は「休憩」と「ガード成功」時のみ |
| **武器とスキル** | 攻撃力・会心・Break値・レアリティは**武器本体**が持つ。スキルは「やり方」（ヒット数・対象数・会心・Break倍率・ためる/集中）を決め、攻撃力は持たない |
| **スキル種類** | ⚔攻撃（ヒット数・対象数・会心率/倍率・Break倍率）/ ▲ためる（次の攻撃2.3倍）/ ◎集中（次の行動EN0・自身0コスト） |
| **スキル連携** | スキルを特定の順番で出すと**追撃**が発生（ためる→攻撃／集中→攻撃／別系統の攻撃を連続）。1手目を出すと、連携になる次のスキルを持つ武器カードが**金色に光って通知**される（`COMBOS`＝`src/game/data.ts`） |
| **会心** | 武器の会心率＋スキルの会心率で発生。会心ダメージは1.5倍（「会心2倍」等のスキルで倍率変化） |
| **武器相性** | 敵の種別（甲殻種・霊体種・飛翔種）ごとに弱点武器があり、弱点ヒットでダメージ増加 |
| **攻撃カウント** | 各敵の頭上に攻撃までのカウントを表示。**プレイヤーが行動するたびに1減り**、0になると予兆「!」が出て少し溜めてから攻撃（この間がガードタイミング） |
| **タイミングガード** | GUARD（軽減0.55＋EN小・地味）/ JUST（軽減0.25＋EN中・中間）/ **PERFECT**（無効化＋HP/EN大回復＋ヒットストップ＋弾き演出＋敵を怯ませ） |
| **ブレイク（気絶）** | 攻撃でゲージ蓄積→満タンで**3ターン**スタン。全ヒット会心（被ダメ増）、敵は攻撃不能。3ターン後に元の待ちへ復帰 |
| **ウェーブ制** | 1ステージ＝**3連戦**。HP/ENを引き継いで戦い抜き、**3戦目はボス**（強敵）。途中で敗北すると入手はなし |
| **複数敵** | タップ/`T`キーや左右矢印で対象選択。カウントが同時に0でも**左の敵から順番に**1体ずつ攻撃（必ずガードで対応可能） |
| **武器レアリティ** | コモン/アンコモン/レア/エピック/レジェンド/アストラル（虹）。**武器ごとに固定**。スキル数を決める（C/U/R=1、E/L/A=2） |
| **ドロップ** | モンスターごとに「ドロップ武器＋ドロップ率」を設定。撃破時に確率で入手し、**武器レアリティ以下のプールからスキルを抽選**（低レアほど出やすい）。レアリティ色の宝箱が出現 |
| **データ外部化** | 武器=`weapons.json` / スキル=`skills.json` / モンスター=`enemies.json` / ステージ=`stages.json`（すべて `src/game/`）。表を編集する感覚で追加・調整できる |
| **インベントリ** | 入手した武器を格納・付け替え。**不要な武器は削除可能**（装備中は不可・2タップ確認） |
| **キャラアニメ** | 待機中の呼吸・浮遊、登場時のスライドイン、攻撃の踏み込みなどを手続き的に表現 |
| **ドット絵 / サウンド** | コード定義のオリジナルスプライト＋Web Audio合成のBGM・効果音（♪でミュート） |

## ゲームの流れ

1. **タイトル** →「ステージ選択」または「インベントリ」
2. ステージを選んで戦闘 → 勝利でレアリティ抽選の武器がドロップ → インベントリに格納
3. クリアで次ステージ解放。インベントリで強い武器に付け替えて挑戦

## 操作方法

- 武器ボタン（または数字キー `1`〜`3`）：各系統で攻撃（スキルは順番に発動）
- ガード（`Space`）／休憩（`R`）／対象切替（`T`）／敵タップで対象選択

## 開発・実行

技術スタック：TypeScript + HTML5 Canvas + Vite

```bash
npm install
npm run dev      # 開発サーバ（ブラウザで開く）
npm run build    # 本番ビルド（dist/ に出力）
npm run preview  # ビルド結果をプレビュー
```

## ディレクトリ構成

```
src/
  game/
    types.ts    # 型定義
    data.ts     # ゲームデータ（スキル・武器・敵・ステージ・各種パラメータ）※バランス調整はここ
    engine.ts   # 戦闘ロジック（複数敵・ダメージ計算・ガード判定・ブレイク）
    game.ts     # 全体進行（タイトル/ステージ選択/インベントリ/戦闘/結果の状態遷移）
    save.ts     # セーブデータ（インベントリ・装備・進行度）の保存/読込（localStorage）
  render/
    canvas.ts   # Canvas描画
    sprites.ts  # ドット絵スプライト定義（オリジナル）
  audio/
    audio.ts    # Web AudioによるオリジナルBGM/効果音
  main.ts       # 入力・ゲームループ・画面別UI配線
  style.css
```
